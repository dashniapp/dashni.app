Encryption Used

Dashni uses standard HTTPS (TLS 1.2 and TLS 1.3) exclusively through Apple's built-in networking APIs — specifically URLSession / NSURLSession — to communicate with backend services (Supabase). No proprietary, custom, or non-standard encryption algorithms are implemented within the application.

Exemption Basis

This application qualifies for exemption under U.S. Export Administration Regulations (EAR) Section 740.17(b)(1) — encryption incorporated into a product that uses only publicly available encryption and relies entirely on the encryption provided by the underlying operating system (Apple iOS).

Encryption Is NOT Used For

  • Custom or proprietary authentication beyond standard HTTPS
  • Local on-device data storage encryption
  • Custom end-to-end message encryption
  • Any algorithm not accepted as standard by IEEE, IETF, or ITU

Conclusion

Dashni does not implement non-exempt encryption. All network communication is secured exclusively via Apple's OS-level TLS implementation. The app is eligible for the standard encryption exemption and does not require an Export Compliance Registration Number (ERN).
